# Tech Stack and Project Working

## Project Overview

This portfolio is a Next.js App Router application that presents Abhijith B's work, writing, events, and profile journey in a single editorial-style site.

Core goals:
- Keep content easy to update from one source (`src/lib/data.ts`)
- Provide both single-page navigation and dedicated deep-dive routes
- Use SEO and metadata best practices for discoverability
- Render live project ranking data from GitHub in the Projects section

## Primary Tech Stack

- Next.js 16.1.6
- React 19.2.3
- TypeScript 5
- Tailwind CSS 4.1.18

## Libraries Used

Runtime dependencies:
- `@vercel/analytics`: analytics tracking in the global layout
- `framer-motion`: animations for event-related UI
- `react-icons`: social and interface icons

Development dependencies:
- `eslint` + `eslint-config-next`: linting
- `postcss` + `@tailwindcss/postcss`: Tailwind/PostCSS integration
- `typescript`, `@types/node`, `@types/react`, `@types/react-dom`

## Architecture and File Roles

App routing (`src/app`):
- `layout.tsx`: root layout, metadata defaults, global font setup, navbar, analytics
- `page.tsx`: home page composed of section components
- Route pages: `about`, `projects`, `blog`, `releases`, `events`, `hackathons`, `profile/*`
- `sitemap.ts` and `robots.ts`: SEO infrastructure

Presentation (`src/components`):
- `Navbar.tsx`: merged minimal nav model with active-state logic
- `sections/*`: homepage sections (home/about/projects/events/blog/releases/contact)
- `features/*`: reusable domain-specific components for events, hackathons, and profile pages

Content/data (`src/lib`):
- `data.ts`: typed content model (site identity, projects, writing, releases, events, profile)
- `github.ts`: live GitHub data fetching and ranking logic for projects

## How the Project Works

### 1. Layout and rendering flow

1. `src/app/layout.tsx` wraps all pages and mounts:
- global styles from `globals.css`
- global navigation (`Navbar`)
- analytics (`@vercel/analytics/react`)

2. Home route (`src/app/page.tsx`) renders section blocks in sequence:
- Home
- About
- Projects
- Events
- Blog
- Releases
- Contact

3. Dedicated routes render corresponding standalone pages for easier sharing and SEO.

### 2. Content flow

Most content is static and typed in `src/lib/data.ts`.

Benefits:
- easy updates
- type safety
- no hardcoded repeated values across UI

### 3. Projects section (live GitHub functionality)

`src/components/sections/ProjectsSection.tsx` now uses live GitHub data from `src/lib/github.ts`.

Working details:
1. Fetch repositories for user `abhi-jithb` from GitHub REST API.
2. Filter out forked/archived/disabled repos.
3. For each repository, fetch:
- star count
- commit count (from commits endpoint pagination)
- languages (top languages from languages endpoint)
4. Rank repositories by:
- stars (descending)
- commits (descending)
- name (alphabetical tie-break)
5. Render ranked cards with stats and keep `Open repo` action unchanged.

Notes:
- API calls are cached with Next.js revalidation to reduce rate-limit pressure.
- If GitHub data is temporarily unavailable, the UI shows a friendly fallback message.

### 4. Navigation functionality

`Navbar.tsx`:
- merges links into minimal groups (`Work`, `Updates`) plus core anchors
- tracks active section on home via `IntersectionObserver`
- highlights route states for non-home pages
- provides responsive mobile menu with same semantics

### 5. SEO and metadata functionality

- Global metadata defaults in `layout.tsx`
- Route-level metadata in page files
- Open Graph/Twitter cards configured
- Sitemap and robots endpoints generated by App Router files
